## CLI

```sh
$ gleam run -m parrot init
> parrot seems to already be configured in your `gleam.toml`. proceed and override anyways? [y/n]
> what database are you using? [sqlite/postgresql/mysql]
> would you like parrot to download and manage sqlc? [y/n]
> provide the path for the sqlc binary (`system` for default)
```

```sh
$ gleam run -m parrot help
```

```sh
$ gleam run -m parrot generate
> downloading sqlc binary...
> generating gleam code ...
```

```sh
$ PARROT_DEBUG=1 gleam run -m parrot generate
> downloading sqlc binary...
> parsing sql queries ...
> generating gleam code ...
```

```sh
$ gleam run -m parrot generate
> downloading sqlc binary...
> generating gleam code ...

> There was an unforseen error. You can use `PARROT_DEBUG=1` to output additional information
  that will be helpful when trying to help you out.
  Please file an issue here: https://github.com/daniellionel01/parrot/issues
```
